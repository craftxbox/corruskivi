# Unitystation: Server Specific Authentication
This document outlines the authentication process between users, servers, and the server list.   

## Glossary of Nomenclature
- **User**: An individual who wants to connect to a Unitystation server. They have an email and password for login.
- **Client**: The compiled Unity project, considered an unsafe environment as it is controlled and modified by forks.
- **Server**: A server that clients may connect to. It holds an asymmetric key pair to prove its unique identity.
- **Auth Realm**: An authentication server verifies user credentials and issues authentication tokens.
- **Server List**: A server that maintains a list of available servers and their versions.
- **Hub**: The launcher client which users log into, retrieve data from one or more server lists, download client binaries, and handle communication between the client and auth realms. Considered to be the secure environment.
- **Connection Challenge**: A unique string independently generated by both the client and server during the connection process.
- **Scope Token**: A token issued by the auth realm that is specific to one server via its key pair. Used to prove the user's identity to that server.
